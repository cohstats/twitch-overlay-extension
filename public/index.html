You have two options for using this boilerplate when developing Twitch Extensions:
  1. Use the Twitch Developer Rig and follow the instructions in the included README. This is going to be the easiest solution, but does not support all Twitch functions, including Bits.
  2. Develop on Twitch when in Local Test, which treats your extension as if it were live. This is going to accurately reflect the realities of production, and supports all extension functions.

Showing Extension on Twitch in Local Test

This process takes a larger amount of time to set up, but as noted above, allows you to test your extension on the full Twitch site.

  1. Now that you have the boilerplate loaded and installed, you'll need two things first.
    • An extension made on the Twitch Developer Site. If you're unsure of what kind of extension view to use, select them all as you can turn them off later. Out of the box, the boilerplate supports all views via multiple entry points.
    • The extension installed on your own channel. This can be done in the "Invite Only" section of the Extension Manager, where you'll find your extension listed.
  2. Once you've installed your extension, you'll need to activate the extension and add it to any of the available slots: Panel, Component, or Overlay. Do note that Component or Overlay extensions require you to be live when testing.
  3. Go to your channel on Twitch and you'll have to click on "Accept" on the extension. It should load.
  4. If it doesn't load, don't fret! Simply visit the URL for the view (https://localhost:8080/panel.html for a panel view, for example) and accept the certificate. Go back to your channel page on Twitch and you'll be good to go!

Now that you're up and running, once you're in a good finished state, see the README on building production files and uploading them to the Developer site.
